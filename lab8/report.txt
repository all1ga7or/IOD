ЗВІТ З ЛАБОРАТОРНОЇ РОБОТИ №8
Тема: Визначення максимального прискорення систем за допомогою законів Амдала

Мета роботи: Навчитися розраховувати потенційне прискорення багатопроцесорних систем, аналізувати вплив послідовної частки коду на ефективність паралелізму та виявляти логічні несумісності в параметрах системи.

Варіант №15
Вихідні дані (Таблиця 2):
- Частка паралельних обчислень (1 - β): 0.65
- Задане прискорення для системи (S): 14
- Цільовий діапазон прискорення: від 65% до 90% від граничного (Smax)

---

1. ТЕОРЕТИЧНІ РОЗРАХУНКИ

Спершу визначимо частку послідовних обчислень (β):
β = 1 - (1 - β) = 1 - 0.65 = 0.35

Визначимо максимально можливе (граничне) прискорення згідно з 1-м законом Амдала:
Smax = 1 / β = 1 / 0.35 ≈ 2.857

Тепер спробуємо розрахувати кількість процесорів (l), необхідну для досягнення заданого прискорення S=14. Використовуємо формулу:
S = l / (l*β + 1 - β)
14 = l / (l*0.35 + 0.65)
14 * (0.35l + 0.65) = l
4.9l + 9.1 = l
3.9l = -9.1
l ≈ -2.33

Як бачимо, результат від'ємний, що фізично неможливо.

---

2. АНАЛІЗ НЕСУМІСНОСТІ

У ході розрахунків було виявлено серйозну несумісність між часткою паралельних обчислень та бажаним прискоренням.
Причина: Задане прискорення (14) перевищує теоретичний ліміт системи (Smax = 2.857). Навіть при нескінченній кількості процесорів система з 35% послідовного коду не може прискоритися більш ніж у 2.857 раза.

---

3. ПРОПОНОВАНИЙ СУМІСНИЙ ВАРІАНТ

Для того, щоб система функціонувала адекватно, і прискорення 14 було досяжним, необхідно значно зменшити частку послідовного коду.
Запропоноване значення (1 - β) = 0.95 (тобто лише 5% коду є послідовним).

Нові розрахунки:
β = 0.05
Smax = 1 / 0.05 = 20
Тепер S=14 є досяжним. Знайдемо кількість процесорів l:
14 = l / (0.05l + 0.95)
14 * (0.05l + 0.95) = l
0.7l + 13.3 = l
0.3l = 13.3
l = 44.33 => приймаємо l = 45 процесорів.

Знайдемо кількість процесорів для забезпечення цільового діапазону (65%-90% від Smax):
Нижня межа (65% * 20 = 13):
13 = l / (0.05l + 0.95) => 0.65l + 12.35 = l => 0.35l = 12.35 => l ≈ 35.28 => l = 36
Верхня межа (90% * 20 = 18):
18 = l / (0.05l + 0.95) => 0.9l + 17.1 = l => 0.1l = 17.1 => l = 171
